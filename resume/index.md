## 2020/02〜2024/08 [TIPSTAR](https://tipstar.com/)の開発・保守・運用
- メンバー規模: 10〜20人
- 利用技術: Golang, Google Cloud, Kubernetes, Terraform, Docker, Github Actions, CircleCI, Redis

### 内容
- Golang + Echo + OpenAPIを軸にしたREST APIバックエンドAPIサーバーの開発・運用・保守業務に従事
- 携わっていた期間ほぼ全てにおいて、TIPSTARのサービスの根幹である金銭トランザクションの保守・運用・改善業務に従事し、お金を扱うTIPSTARというサービスの信頼性工場に大きく寄与した。
  - 車券の購入は複雑なトランザクションになっており、データ整合性担保の難しさが課題となっていた。そこでCloud Pub/Subで非同期処理を活用して結果整合性を取れるように仕組みを改善。毎月数件単位で発生していた不整合を月平均1件未満に削減することができた。
- 利用する決済システムの移行作業を、決済関連の開発リーダーとして決済システムの開発部署と連携しながら達成。
- 法律(景品表示法)上の制約により施策を発案してからGOサインが出るまでに多大な時間を要することが足枷となっていた。 。そのため還元額を管理するためのテーブルを設計しユーザーへの還元額が上限を超えないような処理を追加した結果、企画が本番サービスに上がる時間の削減および企画立案の自由度が大幅に向上した。
- Twitterへ自分の購入した車券を投稿する機能を付けたいという要望があり、Cloud Runを使い、OGPに必要な画像を構成するAPIを構築した。
- Redis ClusterとgRPCサーバーを作成し、気軽にレースをユーザーが実況することができるチャットサービスを作成した。
- ユーザーへの信頼性を重視し、通信エラーなどで金銭トランザクションに不整合が生じた際のユーザー体験を改善するために働きかけ、リーダシップを発揮しユーザー体験の改善のための開発に従事した。
- 運用当番業務に従事し、緊急時の一次対応・インシデントハンドリングの対応を行なった。また、CSチームと連携して不具合のお問い合わせに対する調査対応ならびに修復対応も十分に経験した。
- 不正利用の調査業務の経験
- 運用tipsをまとめたWebページをGCP上にホスティングし、困った時に対処方法がすぐに確認できる仕組みを整えた

## 2024/09〜2025/07 [TIPSTAR](https://tipstar.com/)のインフラ業務
- メンバー規模: 3〜5人
- 利用技術: Google Cloud, Kubernetes, Terraform, Docker, Github Actions, CircleCI

### 内容
- GKE上で動いているワークロードのresource requestの値が大雑把で、無駄にリソースを確保しその結果大量の無駄なノードが立ってしまっていた。
  - 開発環境とステージング環境において適切にリソースの再割り振りを行った結果、一年あたり1,000万円以上のコスト削減を達成。
- Google内で提供が終了するサービスの移行作業を遂行
  - Container RegistryからArtifact Registry, DynamiclinkからAppsflyer Onelink
- Redisのコストカットのために、負荷のかからない環境はインスタンス単位ではなくDB単位にしたり、DBも共有するように工夫するといった対応を実施
- GKE上でシークレット管理のために初期から使っていたsealed-secretsをexternal secretsへ移行する作業を実施
- Github Actionsのrunnerを可能な限りarm64マシンへ移行し、コストカットを実現
- 公営競技(競輪)の一大イベントに合わせ、必要なリソースを計算してスケーリングし、イベントでのサーバーダウンを防止しつつコストもかかりすぎないように対処した
- Google Cloudのアラートポリシーなどの整備を行い、1件エラーログが出ただけで即時対応・補償が必要な事例の対処スピードを向上することに繋がった
- IAMの整理対応の実施(過剰な権限の削除等)
- Google サポートと積極的に不明点を解決にコミュニケーションをとった
